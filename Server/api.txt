클라이언트와 HTTP로 통신합니다.
HTTP 본문의 형식은 JSON입니다.
특별히 표기된 경우를 제외하면 Content-Type 헤더는 application/json입니다.


<회원가입>
	POST /api/user

	헤더
		없음

	파라미터
		이름        타입     설명
		account     문자열   새로 생성할 계정. 최대 길이 30.
		password    문자열   새 계정의 비밀번호. 최대 길이 30.
		nickname    문자열   새 계정의 닉네임. 최대 길이 30.

	응답
		이름        타입     설명
		id      정수     유저를 가리키는 고유한 값

	요청 예시
		POST /api/user
		Content-Type: application/json

		{
			"account": "hello",
			"password": "world123",
			"nickname": "john"
		}

	응답 예시
		{
			"id": 11
		}

		}

<게시글 목록 요청, 검색 요청>
	GET /api/post
		
	헤더
		없음
		
	파라미터
		이름        타입        설명
		category    문자열      카테고리 이름. 이 카테고리에 속한 게시글들만 요청함. 
		                        이 파라미터가 없으면 전체 게시글 목록이 반환됨.
		cursor      문자열      목록의 다음 부분을 요청할 때 사용하는 파라미터.
		                        이 파라미터가 없으면 목록의 제일 앞 부분이 반환됨.
		
	응답
		이름                      타입        설명 
		cursor                    정수        다음 목록 요청 시 사용해야 하는 cursor 값.
		list                      배열        객체 배열. 길이 50.
		list[0].id                정수        게시글의 고유 번호.
		list[0].title             문자열      게시글 제목.
		list[0].creationTime      문자열      게시글 생성 시각. UTC.
		list[0].viewCount         정수        조회수.
		list[0].likeCount         정수        좋아요 수.
		list[0].dislikeCount      정수        싫어요 수.
		list[0].category          문자열      게시글이 속한 카테고리 이름.
		list[0].user              객체        
		list[0].user.id           정수        게시글 작성자의 고유 번호.
		list[0].user.nickname     문자열      게시글 작성자의 닉네임.
		
	요청 예시
		GET /api/post?category=인증&cursor=123

	응답 예시
		{
			"cursor": 173,
			"list": [
				{
					"id": 123,
					"title": "광어 인증",
					"creationTime": "2022-10-09 12:00:00.000Z",
					"viewCount": 123,
					"likeCount": 12,
					"dislikeCount": 3,
					"category": "인증",
					"user": {
						"id": 12,
						"nickname": "john"
					}
				},
				...,
				{
					"id": 172,
					...
				}
			]
		}
